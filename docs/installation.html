Unpacking and basic setup.

You're reading this so you've already unpacked the compressed archive. Now move the contents to wherever you want them to be and issue the following commands from the root of the package:

chmod 777 data/templates_c data/cache

Apache server configuration

AGCDR requires a separate virtual host. The following virtual host configuration is suitable. Change "*:port" to whichever IP address and port number you wish to use and change "/path/to/public/dir" to point to the "public" sub-directory of the un-packed application.

<VirtualHost *:port>

        DocumentRoot /path/to/public/dir

        <Directory /path/to/public/dir/>
                Options SymLinksIfOwnerMatch
                AllowOverride All
                Order deny,allow
                allow from all
        </Directory>

	RewriteCond %{REQUEST_URI} !^/images/
	RewriteCond %{REQUEST_URI} !^/libraries/
        RewriteCond %{REQUEST_URI} !^/css
        RewriteCond %{REQUEST_URI} !^/js
	RewriteCond %{REQUEST_URI} !^/Favicon.ico
        RewriteRule ^(.*)$ /index.php?route=$1

        ErrorLog /path/to/error.log
        LogLevel debug
        CustomLog /path/to/access.log combined

</VirtualHost>

Given that this software exposes potentially sensitive information regarding the usage of your PBX you will most likely wish to lock down your virtual host by IP or by using HTTP auth. AGCDR has no authentication or access control methods of its own.

Configuration

All configuration is done in application/config.php. Edit this file and change the following as a minimum:

  1. Change the "LIVE_URL" constant to match the virtual host under which you have installed AGCDR.
  2. Add the details of at least one database server which hosts the CDR data generated by the cdr_mysql Asterisk module. Complete the fields as shown in the demonstration.
